## The per-civ workflow (agreed)

1. **Evaluate** the next civ → propose chapter count + event count. Target **10–15 events per chapter, don't force it** (thematic chapters lighter; war/political chapters heavier). Adapt structure to the civilization (WRITING-RULES): dense modern/dynastic political TLs ran 12–15 chapters; a classical/archaeological civ fits the standard ~8-chapter shape with a discovery/rediscovery frame. Balance the 8 categories where the subject allows.
2. **Get the numbers OK'd by the user** before writing.
3. **Run the full pipeline to completion** (do not stop between steps):
   - Pull/expand `reference-data/{tlId}.json` to the agreed event count. If a v1 source exists at `~/projects/personal/timeline/src/data/{tlId}.json`, preserve every v1 event ID verbatim (reuse the v1 objects, append new ones; you MAY correct clearly-corrupted v1 `year` fields). **If no v1 source exists (the case for dai-viet — see below), build the reference data from scratch** with a clean `evt-{tl}-` id scheme, 6 spans, full metadata; no IDs to preserve.
   - Write the narrative `narratives/{tlId}.md` (front-matter title + Goal + Status; Chapter Outline table; a names/pronunciation note; a "first"/uncertainty note; then chapters; apply `WRITING-RULES.md`). Chapter headers are `# Chapter N — Title` (single `#`); section breaks are `---`.
   - Register `'{tlId}.md': '{tlId}'` in `scripts/parse-narratives.ts` `NARRATIVE_FILES` (after the last entry, before the closing `}`).
   - Curate `content/.event-links-{tlId}.json`, `.glossary-links-{tlId}.json`, `.cross-links-{tlId}.json` with a throwaway Python builder that splits the md on `^# Chapter (\d+) ` and regex-checks every matchText is a `\b`-bounded (case-insensitive) substring of its chapter's raw markdown; iterate until 0 invalid, then delete the script. Gotchas that bit khmer: `**bold**` *between two words of a phrase* breaks the match (keep matchText inside one bold span or fully plain); `*italic*` does the same (e.g. `a *naval* battle` breaks "naval battle"); put each event in the chapter whose prose actually contains the phrase. Grep target-TL outline tables so cross-link target chapters are real and topic-aligned.
   - Write `narratives/{tlId}.summaries.json` (8 bullets/chapter, 25–40 words, one sentence each, informal voice, plain text only, phrasing mirroring the narrative so links auto-inject).
   - Write `map-prompts/{tlId}.md`: header; a **GLOBAL RULES** preamble (the exact CRITICAL RULES block + chapter-title directive + a per-TL `{TL} ORIENTATION RULES` block); then per chapter `## Chapter N — Title` **repeating the full CRITICAL RULES block + chapter-title directive + "Apply the {TL} ORIENTATION RULES above." + a map description + "5 sites, name-only labels:" list + "4 region labels (gray, orientation only):" list + one annotation + "Landscape orientation, match series style."** Generate: `node --env-file=.env.local scripts/generate-maps.mjs {tlId}`; **spot-audit at least the trickiest 4 thumbnails, ideally all** (the khmer run had 1 of 10 hallucinate an invented river label — viewing all 10 caught it); regen bad chapters with `--chapter N` after `rm`-ing the bad PNG (the script skips existing); if a chapter invents a label, add a targeted negative to that chapter's spec ("Do NOT label any river other than X; there is no river named 'Y'"). Then `node scripts/optimize-maps.mjs` (only the new TL has PNGs, so the whole-dir walk is targeted — verify with `find public/maps -name '*.png'`; it deletes PNG originals, webp is the only copy).
   - 5-persona audit (manual procedure in `.claude/skills/audit-narrative.md`): spawn 4 general-purpose subagents A/B/D/E in parallel (each reads the file itself and Writes its own `audits/{tlId}-persona-{a,b,d,e}.md`, replies only "done"), then merge to `audits/{tlId}.audit.md` (order D, E, A, B; use a Python assembler to avoid loading ~95KB into context), delete the persona files. Apply must-fix factual/citation corrections; record should-fix/deferred here.
   - `npm run parse`; confirm `Parsing {tlId}.md` line shows the right chapter/event/map counts and there are **0 khmer-style warnings for the TL** (filter on the tlId; other TLs' backward-cross-link warnings are pre-existing noise) and no `No summaries found for {tlId}`.
   - Flip `hasContent: true` in `src/lib/navigator-tls.ts` (the entry already exists; just append `, hasContent: true` before the closing `}`).
   - Commit + push. `content/` is gitignored: **`git add -f` the three `content/.{event,glossary,cross}-links-{tlId}.json`**. `content/{tlId}.json` and `public/offline/{tlId}.manifest.json` are gitignored regenerated artifacts — do NOT commit them. `public/search-index.json` IS tracked — commit it. Commit normally: `narratives/{tlId}.md`, `narratives/{tlId}.summaries.json`, `reference-data/{tlId}.json`, `audits/{tlId}.audit.md`, `map-prompts/{tlId}.md`, `public/maps/{tlId}/` (webp), `scripts/parse-narratives.ts`, `src/lib/navigator-tls.ts`.
4. **Overwrite this HANDOFF.md** with the next civ's pre-approved plan + any new deferred items.
