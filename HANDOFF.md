## The per-civ workflow (agreed)

1. **Evaluate** the next civ → build a **narrative-movement map**: list the distinct dramatic movements the civilization's story has (rise / crisis / transformation / collapse / afterlife …). **Chapter count = number of movements** — NOT a civ-type template ("~8–10 for classical"), NOT "take the heavier option," and with **no artificial cap**: a story that needs 22 chapters gets 22. Event target **10–15 per chapter** is enforced by `lint-density.ts` (new civs zero-tolerance). Balance the 8 categories where the subject allows.
2. **Get the chapter list + one-line throughline per chapter OK'd by the user** before writing. This approval is a hard gate — it holds **even in no-stop mode** (no-stop may proceed on everything else, but never auto-pick the chapter count). The retired "take the heavier option" convention no longer applies.
3. **Run the full pipeline to completion** (do not stop between steps):
   - `reference-data/{tlId}.json`. If a v1 source exists at `~/projects/personal/timeline/src/data/{tlId}.json`, preserve every v1 event ID verbatim (reuse v1 objects, append new ones; may fix clearly-corrupted v1 `year` fields). **If no v1 source exists, build from scratch** with a clean `evt-{tl}-` id scheme, 6 spans, full metadata `{id,label,year,(endYear),description,wikiSlug,tier,tags:[],category}`. 8 categories (exact keys): rulers, religion, architecture, science, wars, trade, writing, people. A throwaway Python builder that asserts unique ids and prints the category histogram is the clean way (≈10–13 events/chapter; 0–2 data-only events with no inline link is normal).
   - Write `narratives/{tlId}.md`: front-matter `# {Title} — {Subtitle}` + `**Goal:**` + `**Status:**`; `## Chapter Outline (N chapters)` table `| # | Chapter | Era | Key topics |`; a names/pronunciation note; a "first"/uncertainty note; then chapters as `# Chapter N — Title` (single `#`), section breaks `---`. Apply `WRITING-RULES.md` (miles-first; parens not em-dashes for definitions/asides; inline-define every term incl. other civs; pronunciations on hard names; cross-civ comparisons at "firsts"; hedge contested claims; Ch1 forward-refs inline-defined; introduce every source with a one-line who/when/why; don't repeat an analytical move >2× without differentiating).
   - Register `'{tlId}.md': '{tlId}'` in `scripts/parse-narratives.ts` `NARRATIVE_FILES` (after the last entry, before the closing `}` — **the `majapahit.md` line is now the last entry**).
   - Curate `content/.event-links-{tlId}.json`, `.glossary-links-{tlId}.json`, `.cross-links-{tlId}.json` (dicts keyed by chapter-number string). The parser matches with `new RegExp('\\b('+escaped+')\\b','i')` against the **raw chapter markdown**, applied **cross → event → glossary**, longest-matchText first, each replaced once outside existing `<a>`. **matchText rules:** must be a verbatim plain substring of that chapter; **no `**`/`*`/`[` `]` inside the span** (a single `**bolded term**` is fine because the `\b` lands between `*` and the word, but a phrase that *spans* a `**` breaks); no leading/trailing punctuation; **must not be a superset of, or duplicate, an existing event/glossary/cross matchText in the same chapter** (cross is applied first and will steal the span — this bit the backward pass; see below). Build with a throwaway Python builder + a **Node validator that mirrors `replaceOutsideAnchors` exactly** (split on `<a>`, escape, `\b(...)\b` i, cross→event→glossary, dedup) and iterate to **0 invalid**, then delete both scripts. ~10–14 event / ~8–12 glossary / ~14–18 cross per TL.
   - Write `narratives/{tlId}.summaries.json` (list, one obj/chapter: `chapter`,`title`,`dateRange`,`bullets`). 8 bullets/chapter, 25–40 words, one sentence, informal voice, **plain text only** (no markdown), phrasing mirroring the narrative so links auto-inject.
   - Write `map-prompts/{tlId}.md` via a throwaway Python assembler: header; GLOBAL RULES preamble (exact CRITICAL RULES block + chapter-title directive + a per-TL `{TL} ORIENTATION RULES` block); then per chapter `## Chapter N — Title` repeating the full CRITICAL RULES block + chapter-title directive + "Apply the {TL} ORIENTATION RULES above." + map description + "5 sites, name-only labels:" + "4 region labels (gray, orientation only):" + one annotation + "Landscape orientation, match series style." Generate: `node --env-file=.env.local scripts/generate-maps.mjs {tlId}` (uses `GOOGLE_API_KEY` in `.env.local`, model gemini-3-pro-image-preview). **Run map gen as a background Bash command and spawn the 4 audit agents in the same turn** — both finish while you wait. **Spot-audit ALL thumbnails** (downscale with `sips -Z 760 … --out /tmp/...` and Read them in batches of 3–4 to keep context small; do NOT inline full-size PNGs — they crash sessions). Regen bad chapters: `rm` the bad PNG then `--chapter N`. Then `node scripts/optimize-maps.mjs` (PNG→WebP q85, deletes PNGs; verify only the new TL has PNGs first with `find public/maps -name '*.png'`). **majapahit's 10 maps were all clean first try — no regen needed; this is achievable, the orientation-rules preamble works.**
   - 5-persona audit (`.claude/skills/audit-narrative.md`): spawn 4 general-purpose subagents A (Newcomer)/B (Skeptic)/D (Story Editor)/E (Cross-Cultural) in parallel; each reads the file itself and Writes `audits/{tlId}-persona-{a,b,d,e}.md`, replies only "done". Merge with a Python assembler into `audits/{tlId}.audit.md` in order **D, E, A, B**, delete persona files, then append an **"Editorial triage & fixes applied"** section recording must-fixes done + deferred (see majapahit.audit.md for the format). Lower-priority Persona-A/B/D polish may still be triaged/deferred with a recorded reason. **Re-run `npm run lint:links --strict` after narrative edits** (edits can break or move matchText).
   - **Backward cross-cultural pass** (Persona E Part 2): add terse cross-link JSON entries into the shipped reference TLs' `content/.cross-links-{ref}.json` pointing at the new TL, using **verbatim plain phrases** from the right reference chapter, each **non-overlapping with existing links there**. Validate with the real parser (next step), not just the mirror.
   - `npm run parse`; confirm the `Parsing {tlId}.md` line shows the right chapter/event/map counts, **zero `⚠` warnings whose text mentions {tlId} or any backward-pass reference TL**, and no `No summaries found for {tlId}`.    - Flip `hasContent: true` in `src/lib/navigator-tls.ts` (entry already exists; append `, hasContent: true` before the closing `}`). You must `Read` the file region before `Edit` (harness loses track after a `grep`).
   - Commit + push. `content/` is gitignored: **`git add -f`** the new TL's three `content/.{event,glossary,cross}-links-{tlId}.json` **and every `content/.cross-links-{ref}.json` you touched in the backward pass**. `content/{tlId}.json` and `public/offline/*.manifest.json` are gitignored regenerated artifacts — do NOT commit them. `public/search-index.json` IS tracked — commit it.
