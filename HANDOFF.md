## The per-civ workflow (agreed)

1. **Evaluate** the next civ → propose chapter count + event count. Target **10–15 events per chapter, don't force it** (thematic chapters lighter; war/political chapters heavier). Adapt structure to the civilization (WRITING-RULES): dense modern political TLs ran 13–15 chapters; a classical/archaeological civ fits the standard ~8-chapter shape with a discovery/rediscovery frame. Balance the 8 categories where the subject allows.
2. **Get the numbers OK'd by the user** before writing.
3. **Run the full pipeline to completion** (do not stop between steps):
   - Pull/expand `reference-data/{tlId}.json` to the agreed event count (preserve every v1 event ID verbatim — reuse the v1 objects, append new ones; you MAY correct clearly-corrupted v1 `year` fields, as was done for rise-of-china's 7 garbled years, but keep IDs/labels/text).
   - Write the narrative `narratives/{tlId}.md` (outline table at top, notes on names/contested-numbers/perspective, then chapters; apply `WRITING-RULES.md`).
   - Register `'{tlId}.md': '{tlId}'` in `scripts/parse-narratives.ts` `NARRATIVE_FILES`.
   - Curate `content/.event-links-{tlId}.json`, `.glossary-links-{tlId}.json`, `.cross-links-{tlId}.json` with a throwaway Python validator that regex-checks every matchText is a `\b`-bounded substring of its chapter's raw markdown; delete the script after the files validate. Grep the target TL outline tables so cross-link target chapters are real.
   - Write `narratives/{tlId}.summaries.json` (8 bullets/chapter, informal voice, phrasing mirroring the narrative so links auto-inject).
   - Write `map-prompts/{tlId}.md` (global CRITICAL RULES + a per-TL orientation preamble + per-chapter lean spec: 5 sites, 4 region labels, one annotation arrow; repeat the CRITICAL RULES block per chapter — the validated pattern). Generate: `node --env-file=.env.local scripts/generate-maps.mjs {tlId}`; spot-audit the trickiest 4 thumbnails; regen bad chapters with `--chapter N`; `node scripts/optimize-maps.mjs` (only the new TL has PNGs, so the whole-dir walk is effectively targeted — verify with `find public/maps -name '*.png'`).
   - 5-persona audit (manual procedure in `.claude/skills/audit-narrative.md`): spawn 4 general-purpose subagents A/B/D/E in parallel, each writes `audits/{tlId}-persona-{a,b,d,e}.md`, then merge to `audits/{tlId}.audit.md` (order D, E, A, B), delete the persona files. Apply must-fix factual/citation corrections; record should-fix/deferred here.
   - `npm run parse` until 0 warnings for the TL (other TLs' pre-existing backward-cross-link warnings are noise — filter on the tlId). **Re-run parse after generating+optimizing maps** so the offline manifest shows "15 maps" not "0 maps".
   - Flip `hasContent: true` in `src/lib/navigator-tls.ts`.
   - Commit + push (`git add -f` the gitignored `content/.{event,glossary,cross}-links-{tlId}.json`; `reference-data/{tlId}.json` and `audits/{tlId}.audit.md` are tracked normally).
4. **Overwrite this HANDOFF.md** with the next civ's pre-approved plan + any new deferred items.

Gotchas: `content/` is gitignored — `git add -f` the three curation files. The optimize-maps script **deletes** PNG originals (webp is the only copy). matchText is matched case-insensitively with `\b(escaped)\b` against the **raw markdown**: `**bold**` does not break a match, but `**` *between two words of the phrase* does (e.g. `the **Term**` breaks "the Term") — keep matchText inside one bold span or in plain text; parentheses/trailing punctuation in matchText also break it. `cd` in a Bash call persists for the session — use absolute paths or cd back.
